# 就业指导

就业指导：利用有限的时间，最大限度的提升各个环节的表现，从而提升面试的的通过率。

## 求职过程

1. 简历：制作 + 投递
2. 笔试、面试：项目、场景、纯技术、非技术问题
3. 谈薪

## 简历

- 简历的好坏会影响简历通过率
- 简历的的夸张程度会影响面试难度

**找到一个平衡使综合通过率最高：保证简历通过率的同时，要能够让面试表现和简历基本匹配。**

### 简历优化

1. 工作经历:
   - 30 岁以内 2 份为宜
   - 30 岁以上 3 份为宜
   - 工作数量太多：合并除最后一份以外的经历

2. 工作年限：
   - 与年龄相符；工作年限不满 3 年尽量凑满；项目需要匹配工作年限
   - 当前空窗期不超过 3 个月：延长除了最后一段的工作经历
   - 当前空窗期超过 3 个月：造一份工作经历

> 工作经历不够：[前往](https://shiming.gsxt.gov.cn/corp-query-homepage.html)查找公司
> 登录 - 信息公告 - 撤销登记（备案）公告 - 区域筛选

### 简历要点

1. 个人信息
2. 求职意向
3. 技术栈
4. 教育经历
5. 个人优势 / 个人评价
6. 工作经历
7. 项目经历

#### 个人信息

- 工作经验：3 年以上
- 电话号码：xxx - xxxx - xxxx
- 教育经历：只写高等教育
- 如果能起正向作用则可以添加博客、git 主页

#### 求职意向

- 意向城市
- 意向岗位：前端开发工程师
- 期望薪资
- 入职时间

#### 个人优势 / 评价

选择写优势还是评价？

- 优势：为业界做出贡献，为公司降本增效，管理、架构经验；能让人眼前一亮，数量大于等于 5，书写位置靠前。
- 评价：侧重描述你是怎样的人；书写位置放在最后，不要书写无效评价（_大话、空化、不可量化_）；尽量和技术相关并能够量化。

#### 工作经历

工作职责：按角色写，不要写具体工作细节

工作成绩：有成绩才写，不同角色侧重点不同，不超过 5 条。

- 有获奖或降本增效（_内部框架 / 库_）
- 不要写具体工作细节
- 不要和项目亮点混淆

#### 项目经历

- 数量： 3 - 5 个 （_有商业价值_）
- 面试时：业务 --> 难点亮点（_根据简历问_）
- **不在简历的问题：弱化防御并引导到准备好的**

项目名称：智能、云、平台（_避免：管理系统_）

担当角色：**按角色写职责**

项目描述：不超过三行，描述项目背景和主要功能、组成模块

技术栈：技术关键名词，只写框架和库级别，统一单词大小写风格

项目职责：只描述做的事情，不写具体实现细节

项目业绩：落地效果，宽泛且有具体数据支撑

#### 项目难点

面试问题：

1. 解决过什么问题？
2. 印象深刻的点？

精心准备：**最近**两个项目是否每个项目都有两三个难点**可说可写**？

- 能够抓住眼球
- 能够表述清楚

::: info STAR 法则

- Situation（_情景_）：描述项目背景和面临挑战，简要说明项目初始状态和所处环境
- Task（_任务_）：说明在项目中负责的具体任务，明确要解决的问题或实现的目标
- Action（_行动_）：详细描述裁去的具体行动和策略，包括如何组织、计划、执行
- Result（_结果_）：展示行动带来的具体成果

:::

## 简历投递

### 打招呼

开场 + 匹配岗位充分理由（_针对性_） + 要回复 + 祝福

> 您好，我对贵公司正在招聘的前端开发岗位非常感兴趣，我有过3年前端开发经验，参与过多个大型项目；非常期望能和贵公司做进一步沟通了解，期待您的回复，祝您工作愉快！

已读不回：

- 隔两三天发
- 缩小时间窗口，要回复，祝福

### 面试邀约

- 留出足够准备时间：2 天以上
- 多渠道了解公司
- 寻找归宿感
- 针对性技术准备

面试前心态调整：

- 战略上的藐视，战术上的重视
- 不要不企业想象成敌人，不要站到对立面，面试是交流，是互相了解，不是打擂台

## 面试

### 面试表现力

面试是求职者不断的向面试官传递信息，这些信息的总和决定了求职者的整体形象；信息包括内容、音量、语气语调、肢体语言、表达节奏、外在形象...

==除内容本身外，所有的表现力决定了面试画风的走向==

**不要尝试变成另外一个人，而是找到自己最好的状态。**

### 面试心理

基本原则：留下一个好印象

1. 对知识盲区合理看待
   - 记不住每一个点
   - 无法保证每个细节
   - 工作学习有一定矛盾
   - 学习有主次之分
2. 对技术的热情（_有热情_）
   - 混迹技术社区
   - 喜欢研究新技术
   - 对代码质量执着
   - 喜欢交流技术
   - 看到特别的效果想要实现
3. 对**这份**工作有向往
   - 企业某些特质吸引
   - 企业特质契合个人特质
4. 向往成为面试官的同事
   - 面试官特质吸引

问到一个简单问题不会：

> 很抱歉面试官，这个问题我在工作中确实没遇到过，这一块还得再看看，有一个关于学习的问题请教你，您怎么在工作中抽出时间学习？

问道一个复杂问题不会：

> 这是个好问题，但我确实没想过，一个很值得研究的问题，下来研究一下，以后有机会向您请教。

非技术问题：尽力而为、保持表达结构、注入个人语言风格、强化个人优势

### 自我介绍

基本原则：

1. ==自然==
2. **让面试官认识到自己的能力**：通过各种方面展现自己与技术有关的能力
3. **让面试官认识到你的意愿**：能不能？
4. 让面试官感到亲切
5. 引导面试官的后续问题

> 中低端岗位：开场 + 意愿 + 能力 + 夸赞 + 问题
>
> 面试官您好，我叫 xxx，非常感谢能给我这次面试机会，我目前有 3 年前端开发经验，之前主要是在国企做苏周到平台的自研产品，也会做一些政府的信息化项目，主要技术栈是 vue 微信小程序这些，然后平时工作中也会注重代码的复用性，封装一些组件。我看咱们公司也是有自己自研的业务线，也是比较感兴趣，希望有机会能够加入。您看你还有什么想了解的？

### 离职原因

==不能是能力问题；不能是态度问题。==**一定不能让面试官感受到压力和不安**

1. 不能对上家有戾气，不能倒苦水
2. 不能出现不负责的行为
3. 对上家公司表现出理解
4. 上家公司的问题是这家公司不可能有的（包括将来）

建议方面：

- 不可抗力：业务线被砍、公司注销、公司搬迁（年纪小慎用）
- 价值观：同事关系复杂、官僚气息重、歪风邪气

> 表达结构：(肯定 + 否定) + 理解 + 期望
>
> 这家公司本来一开始挺好的，就是后来对于我个人发展来说太受限了，公司很多人都是混日子，项目是能拖就拖，应付了事，好多问题我提出来，就算我处理完了，也没人审核。思索再三，不适合这样的工作氛围就离职了。

### 是否接受加班

先肯定，在给期望，再理解

> 加班很正常，没问题
> 加班还是希望能有加班工资，也希望没有无意义的加班。
> 公司有困难，有压力一起抗，没问题。

### 期望薪资

> 想好底线，尽量不要说，说的时候抬高底线

弱势：我更看重发展，薪资只要不是低于行业平均水平，都可以接受

强势：我也面了另外两家公司，其中一家能给我 xxx 另一家还在看意向，我还是非常想来咱们公司的，如果能给到 xxx 的话，我就可以接受。

### 缺点

有好一定有坏；谈优点，就要有缺陷，谈缺陷要聊到优点。

对自我有认知，对问题有办法。

> 我是一个急性子，虽然给我带来一些优势，如行动能力强。另一方面，很多事情也因为着急导致各种失误。
> 这些年我也通过一些方法来改善这一点：做计划，多检查
> 性格难改变，但工作中已经可以避免这类错误。

### 职业规划

表达目标：

1. 学习有方向、有依靠
2. 工作有动力、有目标

- 主线：技术线
- 支线：非技术线

业务开发 --> 架构基建 --> 前端 leader --> 系统架构师

没有从事过架构岗：目标首先成为前端架构师，不断积累经验

已经从事过：学习其他技术，听从公司安排

### 同事关系

从企业角度考虑问题

同事要求帮忙写代码：先看企业是否明令禁止；如果禁止，友好拒绝。如果没有，如果不忙可以帮忙，让领导知道。

同事看法不一致：代码分对错，如果有对错之分很好解决，要据理力争。如果没有，可以提出自己的看法，让领导和同事们一起讨论。

### 问面试官

面试没有结束！不能问自卑且愚蠢的问题：

1. 能不能过
2. 能给多少
3. 要不要加班
4. 有哪些福利

可以体现职业素养，可以做人情世故

1. 如果入职，参与哪一块的业务？
2. 将来如果干的好，有没有晋升的可能性？

适当回应：

1. 该业务我也挺感兴趣的
2. 希望能够跟着公司共同发展

### 面试复盘

- 能录音尽量录音
- 分析那些**做的好**，那些**做的不好**
